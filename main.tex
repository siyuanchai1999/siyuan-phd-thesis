\abstract{
Memory is the central resource of modern computing systems, yet the system software layers
that manage it---conceived for simpler, uniform hardware---have failed to keep pace with the
radical evolution of hardware and workloads.
This dissertation identifies three critical manifestations of this mismatch.
First, commodity OS kernels remain hardwired to radix-tree page tables, blocking
experimentation with new translation architectures on terabyte-scale servers.
Second, strict I/O memory protection in virtualized environments incurs 83--95\% throughput
degradation from per-DMA VM exits, forcing production deployments to disable it entirely.
Third, GPU memory oversubscription for AI workloads is unsolved: CUDA Unified Virtual Memory
(UVM) is more than 5$\times$ slower than manual offloading, forcing each framework to implement its own.

This dissertation argues that these bottlenecks arise not from hardware limitations but from
system software layers designed without awareness of the hardware they manage or the workloads
they serve, and that targeted changes to those layers alone suffice to close the gap.

We present three systems.
\emph{Extensible Memory Translation (EMT)} gives Linux an architecture-neutral interface for
memory translation, enabling OS support for diverse hardware translation schemes---including
hash-table-based and flat page table designs---without modifying the kernel for each architecture.
\emph{Virtually Free} redesigns the Linux IOMMU invalidation protocol using one-for-many
invalidations and deferred page freeing to amortize VM exit costs, achieving performance within
1--8\% of an unprotected system---74--147$\times$ better than the state of the art---while
preserving strict security guarantees.
\emph{PyTorch-UVM} integrates a UVM-aware caching allocator and discard-based eviction directly
into PyTorch, enabling practical GPU memory oversubscription for LLM inference without modifying
application code.

Together, these systems demonstrate that making system software layers hardware- and
workload-aware---through targeted changes---eliminates bottlenecks in performance,
security, and extensibility.
}
